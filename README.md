# 全圓佑 Threads 存檔庫

個人 Threads (@wano.o717) 貼文備份整理網站，共 937 篇，依內容自動分類為 10 大類（其中「日常雜談」再細分 6 小類），支援分類篩選＋站內搜尋＋依日期排序。點卡片會開新分頁連回 Threads 原文。

## 分類說明

分類優先順序：只要文章內出現「D-數字」（例如 D-628），不論還有什麼其他關鍵字，一律歸進 Back系列（這是最高優先權規則），其餘規則依序往下比對。

| 分類 | 說明 |
|---|---|
| 🐈‍⬛ Back系列 | 文章內含「D-數字」的回顧特輯（含少數標題誤標成「飯拍D-」「考古幼貓...D-」等，但只要有 D-數字都算這裡） |
| 📔 全圓日記 | 標題「全圓XX日記」的近況記錄（無 D-數字時才算） |
| 🐈‍⬛🐶 圓奎日常 | 開頭是🐈‍⬛🐶標記、圓奎/珉佑相關的內容（無 D-數字時才算） |
| 🕰️ 考古系列 | 標題「考古幼貓」系列（無 D-數字時才算） |
| 🎤 演唱會巡演 | 巡演、見面會、打歌相關 |
| 🎉 生咖應援 | 生咖／應援物相關 |
| 📸 飯拍分享 | 飯拍分享或出清（已合併成同一類） |
| 👔 時尚代言 | 品牌合作、穿搭 |
| 🧶 手作興趣 | 鉤織等手作紀錄 |
| 🎊 紀念重要日 | 生日、週年、入伍退伍 |
| 💭 日常雜談 | 其餘即興心得（不分子分類） |

分類是用關鍵字規則自動判斷的「大致分類」，不是每篇都100%精準，之後想調整規則可以編輯 `data.js` 裡每筆資料的 `cat` 欄位（或提供更新的 CSV 讓我重新產生 `data.js`）。

🐈‍⬛ Back系列 和 📔 全圓日記 這兩個分類可以點右邊的 ▾ 箭頭展開，依「內文提到的日期年份」分年份子分類（例如全圓86日記提到250423，就會歸進2025年；不是用發文日期，是用內容本身講的日期）。有 42 篇 Back系列文章內文完全沒提到日期/年份，會歸進「年份不明」。

## 版面

分類選單改成「右側可收合選單」：點右上角「☰ 分類」按鈕開啟，點分類會自動套用篩選並關閉選單（手機上），桌面版點外側遮罩或✕也可關閉。Back系列／全圓日記右邊各有一個獨立的展開箭頭，可以各自收合年份子分類，跟「選取分類」這個篩選動作是分開的兩件事。

## 部署到 GitHub Pages（跟生咖網頁一樣的流程）

1. 在 GitHub 建一個新 repository（例如 `wonwoo-threads-archive`）
2. 把這 4 個檔案 `index.html`、`style.css`、`script.js`、`data.js` 上傳到 repo 根目錄
3. 到 repo 的 **Settings → Pages**
4. Source 選 `Deploy from a branch`，Branch 選 `main` / `root`，儲存
5. 等 1-2 分鐘，網址會是 `https://<你的帳號>.github.io/<repo名稱>/`

## 之後想新增貼文

如果之後有新貼文想加進來，把新的 Threads 匯出 CSV 給我，我可以重新跑分類、更新 `data.js`，你只要把新的 `data.js` 覆蓋上傳到 GitHub 就好，不用動其他檔案。

## 檔案結構

```
index.html   頁面結構
style.css    樣式（深色系，紫＋薄荷綠應援色）
script.js    篩選／搜尋／排序邏輯
data.js      937 篇貼文資料（自動產生，勿手動大改格式）
```
